# Predicting Rejsekort Journeys

This repository downloads Rejsekort passenger journey data from passagertal.dk and runs daily and monthly forecasting pipelines.

## What changed

The DVC `ingest` stage no longer uses `pipeline/ingest.py`. It now runs:

```bash
node pipeline/download-daily.js && python pipeline/prepare_daily_ingest.py
```

`download-daily.js` performs the browser-based export and writes `pipeline/rejsekort_latest_year_daily_export.xlsx`. `prepare_daily_ingest.py` validates the export and copies it to `Data(update).xlsx`, so the existing preprocessing and modelling stages can continue from the same expected input name.

The monthly stage runs both monthly export scripts, cleans their different Excel structures, combines the results, runs SARIMA, AutoARIMA, Prophet, and BiLSTM where available, and writes a separate monthly trend image.

## Outputs

| Output | Description |
|---|---|
| `Data(update).xlsx` | Latest-year daily export generated by `download-daily.js` |
| `data_cleaned.csv` | Daily cleaned data used by the daily models |
| `output.png` | Daily BiLSTM forecast plot |
| `monthly_cleaned.csv` | Combined cleaned monthly dataset from both monthly download scripts |
| `monthly_forecast.csv` | 12-month monthly forecasts by algorithm |
| `monthly_model_metrics.csv` | Validation metrics for monthly algorithms |
| `monthly_trends.png` | Monthly trend and forecast image |

## Setup

### Python

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### Node / Playwright

```bash
npm install
npx playwright install chromium --with-deps
```

## Run the pipeline

```bash
dvc repro --force
```

Run only the daily ingest:

```bash
dvc repro ingest --force
```

Run only the monthly trend pipeline:

```bash
dvc repro monthly --force
```

## Pipeline stages

| Stage | Command | Main output |
|---|---|---|
| `ingest` | `node pipeline/download-daily.js && python pipeline/prepare_daily_ingest.py` | `Data(update).xlsx` |
| `preprocess` | `python pipeline/preprocessing.py` | `data_cleaned.csv` |
| `selected_arima` | `python pipeline/selected_arima.py` | `sarima_metrics.csv` |
| `autoarima` | `python pipeline/autoarima.py` | `autoarima_metrics.csv` |
| `prophet` | `python pipeline/prophet_model.py` | `prophet_metrics.csv` |
| `bilstm` | `python pipeline/BiLSTM.py` | `output.png` |
| `monthly` | monthly JS downloads + `python pipeline/monthly_forecast.py` | `monthly_trends.png` |

## Notes

The JavaScript download scripts are kept as browser-ingestion scripts and run in headless mode. The data-cleaning and forecasting logic is built around their Excel outputs rather than replacing them with API scraping.
